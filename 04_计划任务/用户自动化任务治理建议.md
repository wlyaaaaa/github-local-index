# 用户自动化计划任务治理建议

更新时间：2026-07-08

这份文档只回答三件事：

1. 哪些计划任务现在要处理。
2. 哪些任务保留，哪些以后可以删。
3. 哪些任务需要隐藏前台窗口，哪些不要隐藏。

本仓库是公开索引仓库，所以这里只写公开摘要。不保存完整计划任务 XML、完整 Action 参数、原始日志、token、私钥、完整 `.env`、OAuth JSON、数据库、截图或完整机器快照。

## 一句话结论

现在不要急着新增任务，也不要急着删除任务。

本轮已经修好两类问题：路径坏掉的任务，以及两个调度器拒绝运行的用户自动化任务；随后把所有应隐藏的用户自动化任务统一收敛为 `wscript.exe + VBS Shell.Run(..., 0, True) + WScript.Quit(exitCode)`。现在不需要新增任务，也没有建议立即删除的任务；后续主要观察两个“被中断”的周备份任务和少数软件自启/旧入口。用户确认部分中断可能来自手动关闭前台窗口，因此 `0xC000013A` 先按人为中断、关机或注销观察，不再默认当作任务设计缺陷。

## 时间策略

这台机器是个人桌面机，不是 24 小时服务器，所以计划任务不应依赖凌晨 03:00/04:00 这类高概率关机窗口。当前最优解是：

- 日常备份和自动推送放在晚间常用窗口，从 19:45 到 23:30 错峰运行。
- 重量较高的周备份拆到周六/周日晚间，不和每天任务挤在同一分钟。
- 所有应补跑的任务保留 `StartWhenAvailable`，机器关机错过后下次可补跑。
- 登录触发只做兜底，并加入 20 到 30 分钟延迟，避免刚开机时同时抢磁盘、网络和前台焦点。
- 清理类任务降到每小时一次，避免 10 分钟级任务频率造成不必要的打扰。

## 前台窗口怎么处理

### 总原则

当前按你的判断收敛为一句话：

> 自动计划任务默认隐藏且零闪；`DEV` 和 `WECHAT` 只作为手动调试时的可见候选，不把定时任务长期改成前台窗口。

这里的“显示/可见候选”只表示排查方式分类，不等于要把计划任务长期改成前台显示。本轮实际改动只做隐藏方向：所有应隐藏的用户自动化任务都已统一为 `wscript.exe` 调用 VBS，VBS 再用 `Shell.Run(..., 0, True)` 隐藏等待子脚本，并用 `WScript.Quit` 把退出码交回任务计划器。

### 显示/可见候选

| 类别 | 任务 | 为什么可以显示 | 当前建议 |
| --- | --- | --- | --- |
| `DEV` | `DevConfigBackup-Local`、`DevConfigBackup-OnUSB`、`DevConfigBackup-Weekly` | 开发配置备份失败时，前台输出有助于看进度、卡点、Git/Drive/U盘错误 | 自动任务保持隐藏；排查时手动前台运行 |
| `WECHAT` | `WeChatBackup-Weekly` | 微信备份耗时、路径和文件占用问题较多，显示窗口有助于判断是否卡住 | 自动任务保持隐藏；排查时手动前台运行 |
| `WECHAT` | `WeChat AutoStart` | 这是 GUI 软件自启，本来就不是后台脚本 | 不强行隐藏；它是否显示由微信自身启动行为决定 |

### 应该隐藏的

这些属于后台任务，当前已经静默：

- 备份任务
- 心跳任务
- 看门狗任务
- 自动同步任务
- 短 PowerShell 检查脚本

但 `DEV` 和 `WECHAT` 按上面的例外处理；其它备份任务仍默认隐藏。

| 类别 | 任务 | 窗口策略 |
| --- | --- | --- |
| AI 模型备份 | `AIModelsBackup-*` | VBS 零闪隐藏，保留退出码 |
| 开发/微信备份 | `DevConfigBackup-*`、`WeChatBackup-Weekly` | 自动任务 VBS 零闪隐藏；调试时手动前台运行 |
| 记忆/配置备份 | `Codex Memory Backup`、`Gemini Memory Backup`、`OpenClaw Memory Backup`、`TimeAudit_DailyBackup`、`RAMDisk_Code_Backup`、`AutoDigitalBackupToH`、`SteamMillenniumConfigSnapshot` | VBS 零闪隐藏，作为后台恢复点任务 |
| 自动同步 | `Scripts_AutoPush`、`OpenClawGateway AutoPush`、`GitHubLocalIndex Refresh` | VBS 零闪隐藏；公开仓库仍需审查，不靠前台窗口保证安全 |
| 服务自启 | `TimeAudit_AutoStart`、`OllamaStable32100` | VBS 零闪隐藏；登录自启不弹 CMD/PowerShell 前台窗口 |
| 看门狗/心跳 | `OpenClaw Heartbeat`、`WeFlow Watchdog`、`TimeAudit_Watchdog`、`TURZX SideScreen`、`OllamaStable32100SelfHeal` | VBS 零闪隐藏；常驻或高频任务不反复弹窗 |
| 轻量检查 | `SunshineRemote-VerifyPath-Daily`、`PinDefaultAudio-Speaker` | VBS 零闪隐藏；只需看返回码和日志 |

优先方式：

- 自动任务入口：统一注册为 `wscript.exe "xxx_hidden.vbs"` 或等价 VBS 包装器。
- VBS 内部：统一使用 `Shell.Run(command, 0, True)`，`0` 表示完全隐藏，`True` 表示等待子进程结束。
- 退出码：VBS 必须 `WScript.Quit(exitCode)`，避免 PowerShell/BAT 失败时被任务计划器误记为成功。
- 手动调试入口：需要看输出时，不改计划任务；直接在终端前台运行对应 `.ps1` / `.bat`。

## 新增任务判断

当前不建议新增新的自动提交或自动推送任务。

只有一个可选项：`GitHubLocalIndex Consistency Check`。如果你只是想定期发现索引漂移，但不写文件、不提交、不推送，可以注册只读一致性检查任务。

可用入口：

- `tools\Refresh-GitHubLocalIndex.ps1 -CheckOnly`
- `tools\Test-GitHubLocalIndexConsistency.ps1 -SkipFetch`

这个检查任务只负责提醒漂移，不负责自动修复。

## 删除任务判断

当前没有建议立即删除的任务。

以后可以删除的候选只有这些：

- `TURZX WeatherFix`
- `TURZX_88inch_AdminStart`
- `TimeAuditEngine`
- `TimeAuditTelemetryEngine_Fixed`
- `OpenClaw Update`

删除前必须同时满足：

- 替代任务已经存在。
- 替代任务最近运行正常。
- 动作路径有效。
- 你确认不再需要旧任务作回滚入口。

备份类任务不要按这个逻辑删。备份任务出错时，优先修复。
