# 用户自动化任务治理建议

更新时间：2026-07-06

本文件是人工治理结论，基于 `用户自动化任务地图.md`、`计划任务健康摘要.md`、`计划任务异常清单.md`、`仓库计划任务建议.md`、只读 `Get-ScheduledTask` 查询、动作路径存在性检查和本地 Git root 归属判断整理。公开仓库只记录摘要，不保存完整任务 XML、完整 Action 参数、原始日志、token、私钥或完整本机快照。

## 总体结论

- 当前 35 个主候选里，绝大多数是用户自己设置的仓库脚本、备份、看门狗、自启或本机恢复任务，不属于 Microsoft 系统任务或普通软件更新器。
- 强制新增任务：暂无。现有仓库大多已有覆盖；`GitHubLocalIndex Consistency Check` 只适合作为可选的只读一致性检查任务，不应自动提交或推送。
- 立即删除任务：暂无。禁用任务先保留作回退证据；确认替代任务稳定后，再清理旧入口。
- 当前最该修复：`AIModelsBackup-*` 动作脚本路径缺失、`PinDefaultAudio-Speaker` 动作脚本路径缺失、`Gemini Memory Backup` 和 `SunshineRemote-VerifyPath-Daily` 被调度器拒绝、`DevConfigBackup-Weekly` 和 `WeChatBackup-Weekly` 被中断。
- 常驻类 `Running / 0x00041301` 不直接等同异常；对 `LibreHardwareMonitor`、`OpenClaw Gateway`、`TURZX SideScreen`、`WeChat AutoStart` 这类任务，应按“正在常驻运行”理解。

## 返回码速记

| 返回码 | 本次解释 | 处理原则 |
| --- | --- | --- |
| `0 / 0x00000000` | 正常完成 | 保留并继续观察 |
| `267009 / 0x00041301` | 任务仍在运行 | 常驻服务通常正常；若非服务类才查卡住 |
| `267011 / 0x00041303` | 尚未运行、无有效完成记录或禁用态信息码 | 结合 `Disabled` 判断，不单独视为失败 |
| `2147946720 / 0x800710E0` | 调度器拒绝或运行条件不满足 | 查权限、登录态、电源条件、已有实例限制 |
| `3221225786 / 0xC000013A` | 进程被中断 | 查关机、注销、超时或手动终止 |
| `1073807364 / 0x40010004` | 非零特殊退出 | 事件/USB/交互触发任务先复跑确认 |
| `4294770688 / 0xFFFD0000` | 本机证据下优先怀疑路径或设备参数问题 | 先修动作路径和设备名 |

## 新增任务判断

当前不建议新增新的自动提交或自动推送计划任务。

`GitHubLocalIndex Refresh` 已覆盖 `E:\GitHub总索引` 的定期刷新场景，用于生成公开索引、同步诊断和计划任务摘要。该任务只能刷新可公开 Markdown 摘要，不应自动 `git commit` 或 `git push`。

可选新增项只有 `GitHubLocalIndex Consistency Check`：如果只想定期发现索引漂移但不写文件，可用 `tools\Refresh-GitHubLocalIndex.ps1 -CheckOnly` 或 `tools\Test-GitHubLocalIndexConsistency.ps1 -SkipFetch` 注册只读检查任务。它不替代人工/Codex 审查，也不自动修复文档漂移。

## 删除任务判断

当前没有建议立即删除的任务。

可列为后续删除候选的任务：

| 候选 | 删除前条件 |
| --- | --- |
| `TURZX WeatherFix`、`TURZX_88inch_AdminStart` | 确认 `E:\TURZX-SideScreen` 新任务稳定替代旧下载目录版本 |
| `TimeAuditEngine`、`TimeAuditTelemetryEngine_Fixed` | 连续一段时间确认 `TimeAudit_AutoStart` 和 `TimeAudit_Watchdog` 覆盖启动与守护 |
| `OpenClaw Update` | 确认 OpenClaw 更新已改为手动或被其他稳定流程接管 |

删除前必须满足：替代任务存在、动作路径有效、最近运行正常、用户不再需要回滚入口。备份类任务只要承载恢复材料，即使有非零返回码，也优先修复而不是删除。

## 前台隐藏策略

- 需要隐藏：后台备份、心跳、看门狗、自动同步和短 PowerShell 脚本。优先使用 `PowerShell -WindowStyle Hidden` 或 `wscript.exe xxx_hidden.vbs` 包装。
- 特别建议补隐藏：`TimeAudit_AutoStart` 仍是 BAT 入口，若登录时闪黑框，建议改成 VBS 包装；`TURZX SideScreen` 若 PowerShell 前台可见，建议改 `-WindowStyle Hidden` 或 VBS 包装；`OllamaStable32100` 若启动时闪窗，也可补隐藏。
- 已经合适：多数 `*_hidden.vbs`、`wscript.exe` 和已带 `-WindowStyle Hidden` 的任务无需再改。
- 不建议盲目隐藏：`WeChat AutoStart`、`LibreHardwareMonitor`、`AIDA64 AutoStart`、TURZX GUI 旧启动这类 GUI/托盘程序。它们是否显示窗口属于软件使用方式，不应只为“静默”强行隐藏。
- 禁用任务无需处理窗口：`OpenClaw Update`、`TimeAuditEngine`、`TimeAuditTelemetryEngine_Fixed`、`TURZX WeatherFix`、`TURZX_88inch_AdminStart` 当前未参与自动运行。

## 公开记录边界

- 可以记录任务名、状态、返回码、仓库归属、动作路径是否存在、是否疑似旧路径、治理建议。
- 不记录完整任务 XML、完整 Action 参数、原始日志、密钥值、token、完整 `.env`、OAuth JSON、数据库、截图或完整机器快照。
- 私有备份仓库可承担恢复材料职责；本公开索引只写结论，不复制敏感内容。
