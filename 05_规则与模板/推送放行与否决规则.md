# 推送放行与否决规则

## 两阶段门禁

```text
decision=block => no write or push
decision!=block && push_decision!=proceed => read-only diagnosis allowed, direct transport blocked
push_decision=proceed => transport conditions only
visibility=PUBLIC => separate publication review of rules, visibility, commits, paths and content
```

`decision` 控制能否写入，`push_decision` 只控制 Git transport readiness；admission V1 不输出 `publication_decision`。PUBLIC 目标必须在 transport 放行后另查项目规则、实际 visibility、候选 commits、paths 与 content。

## 放行

| 条件 | 处理 |
|---|---|
| GitHub 可见性已确认是 `PRIVATE`，且是备份/恢复/个人知识库 | transport 放行后按目标仓库规则发布，包含密钥或私有配置也不默认脱敏 |
| GitHub 可见性为 `PUBLIC`，目标规则和候选 commits、paths、content 均已复审且公开安全 | publication review 可放行 |
| 当前仓库工作区干净、同步证据 live 且仅 ahead | 只满足 transport 候选条件，不代表公开发布授权 |

## 否决

| 条件 | 处理 |
|---|---|
| 公开仓库含真实密钥值、私钥内容、token 值 | 否决原样提交 |
| 公开仓库含未脱敏截图、原始日志、进程列表、绝对路径密集泄露 | 否决原样提交 |
| 工作区混合临时产物、重复副本、生成帧和代码变更 | 否决 `git add .`，要求整理后复审 |

## 关键命令

```powershell
gh repo view OWNER/REPO --json visibility
git status -sb
git log --oneline --decorate --branches --not --remotes=origin
git push origin <branch>
```
