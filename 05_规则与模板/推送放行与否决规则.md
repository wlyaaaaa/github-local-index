# 推送放行与否决规则

本文件是 transport 与 publication 完整矩阵的唯一副本。其他规则和说明只保存摘要与链接，避免安全语义漂移。

## 判断层次

```text
decision=block => insufficient admission evidence; no write or direct transport, read-only diagnosis remains allowed
decision!=block && push_decision!=proceed => direct transport not ready; diagnosis remains allowed
push_decision=proceed => transport conditions only, never publication authorization
visibility=PUBLIC => review fresh visibility, candidate commits, paths and content
```

admission provider 是可选证据源，不是授权机关。模型可以使用当前 `.git`、GitHub metadata、目标项目规则和候选 diff 等等价的新鲜证据；安全结论取决于证据和实际候选结果，不取决于是否运行某条指定命令。

## 完整矩阵

| 层次与条件 | 允许 | 必须停止或补证 |
|---|---|---|
| 用户当前请求未授权外部写入 | 本地读取、诊断及已授权的本地工作 | push、发布、建 PR、部署等外部副作用 |
| repo identity、remote、目标分支或目标仓库存在实质冲突 | 只读调查当前 `.git`、remote 和 owner 证据 | 写入错误工作区或向不明目标 transport |
| admission `decision=block` 或 provider/schema 失败 | read-only diagnosis；使用等价新鲜证据定位原因 | 依赖不充分 admission 证据写入或直接 transport |
| behind、diverged、conflicted、detached 或无可靠 upstream | 只读诊断；按项目语义制定 reconcile/update 方案 | 把状态自动解释成可直接 push |
| dirty/worktree 状态包含用户改动或混合产物 | 显式限定候选文件，必要时隔离 | `git add .`、覆盖用户改动、混合发布 |
| transport 证据充分且目标明确 | 执行本轮已授权的 Git transport | 推断候选内容已经安全或已经获发布授权 |
| visibility 未知、过期或与其他证据冲突 | 查询当前 GitHub visibility | 含敏感内容的发布或按旧可见性猜测 |
| 目标已确认 `PUBLIC`，候选 commits、paths、content 均复审且公开安全 | 按当前授权发布 | 只凭仓库曾经公开、Hook 通过或扫描器零命中放行 |
| `PUBLIC` 候选含真实 key/token/私钥、完整 `.env`/OAuth JSON | 只提交移除或安全重构后的候选 | 原样提交或推送 |
| `PUBLIC` 候选含原始日志/数据库/聊天/健康资料、私密截图、完整 task XML 或未缩减机器快照 | 仅发布必要、缩减且脱敏的结果 | 原始材料进入 Git 历史 |
| `PUBLIC` 候选含可直接滥用的账号、主机或运维细节 | 降低到实现/审计所需的公开安全信息 | 发布可操作的滥用材料 |
| 目标已确认 `PRIVATE` 且用途是备份、恢复、个人知识库或配置快照 | 在本轮授权范围内 preserve exact content，不自动遮盖、截断或占位 | 因形式化脱敏破坏恢复能力；把私密内容复制到公开索引或聊天 |
| 私有目标可能已转为公开 | 重新取得当前 visibility 并按 PUBLIC 复审 | 沿用历史 PRIVATE 结论继续敏感推送 |
| 目标为 `wlyaaaaa/Key` | 允许在受管私有路径 clone，只提交密文与公开安全说明 | 把解密明文、口令或密钥文件写入 clone；创建计划任务；未经授权发布 |

## 结果解释

- `transport ready`：Git 目标和分支状态支持已授权的传输动作。
- `publication safe`：实际候选内容在当前 visibility 和目标规则下可发布。
- `authorization present`：活动用户请求允许对应外部副作用。

三者缺一不可，且不能互相推出。PRIVATE 目标的可信与精确保真也不自动授予 push；PUBLIC 内容审查通过也不能绕过用户授权。

## 工具角色

- `Get-ProjectAdmission.ps1`：身份、worktree、sync 与 transport 的结构化证据；按信息价值使用。
- 目标项目规则、`git diff` / candidate commits / paths：确定真正准备发布的结果。
- `gh repo view ... --json visibility` 或等价 GitHub metadata：确认当前 visibility。
- `Install-GitHook.ps1` 与内容扫描：defense in depth，可拦截显著模式但不能证明安全；只在 bootstrap/repair 或实际复审需要时使用。
- refresh 与 milestone helper：维护索引事实，不是 publication 门禁，也不是普通任务固定步骤。

发现不确定性时应补最能降低该不确定性的证据；不要为了完成 checklist 调用与当前判断无关的工具。
