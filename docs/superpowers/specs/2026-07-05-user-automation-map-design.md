# 用户自动化任务地图设计

更新时间：2026-07-05

## 目标

扩大 Windows 计划任务审计范围，识别所有不像系统软件、而像用户自己配置的自动化任务；为每个任务写出公开可读的用途推测、存在理由、关联仓库或路径、异常和复查点。同时审阅已发现本地 clone 的 GitHub 仓库，推测哪些仓库“应该有计划任务但还没有”。

## 边界

- 本轮只做只读审计和公开文档更新，不创建、不修改、不删除 Windows 计划任务。
- 不导出完整计划任务 XML，不写完整 Action 参数，不复制 token、私钥、完整 `.env`、OAuth JSON 或原始日志。
- 任务用途和新增任务建议属于推测，必须在文档中明确写成“推测”或“建议”。
- 计划任务建议只写用途、频率和理由；真正创建任务需要后续明确授权。

## 用户自动化任务判定

任务满足任一条件时列入：

- 任务路径不在 `\Microsoft\Windows\` 下，且任务名或路径不像普通软件更新器。
- Action 指向 `E:\`、`G:\`、用户目录、Git 仓库、PowerShell、Python、VBS、bat、cmd 或 git。
- 任务名包含备份、同步、自启、看门狗、推送、心跳、监控、AutoPush、Backup、Watchdog、Heartbeat 等语义。
- 任务虽然禁用，但命名或路径显示是用户长期工作流的一部分。

## 输出文档

- `04_计划任务/用户自动化任务地图.md`：按任务列出状态、返回码、关联路径、用途推测、为什么需要、风险或复查点。
- `04_计划任务/仓库计划任务建议.md`：按仓库列出已有覆盖、是否建议新增计划任务、建议频率、理由和风险。

## 仓库建议规则

- 备份/记忆/配置仓库：通常建议每日或登录时备份，但如果已有对应任务覆盖，则不新增。
- Watchdog/服务桥接仓库：若有本地服务或心跳脚本，建议已有或新增看门狗。
- 公开代码/文档/内容仓库：默认不建议自动提交；只建议“定期状态审计”或“生成公开摘要”，避免把临时产物自动推上公开仓库。
- 数据/benchmark/截图类仓库：不建议自动推送原始产物；可建议生成摘要任务。
- 私有知识库：可建议备份任务，但频率应保守，避免大文件频繁同步。
